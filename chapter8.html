Chapter 8

Sorting Algorithms

1 Learning Objectives

  • Understand and implement Bubble, Selection, Insertion, Merge, Quick, and Heap Sort.
  • Analyse best/average/worst case time complexity for each algorithm.
  • Differentiate Stable vs Unstable and In-place vs Not sorts.
  • Understand non-comparison sorts: Counting, Radix, Bucket Sort.
  • Choose the right sorting algorithm for a given real-world scenario.
  • Implement sorting algorithms in Python from scratch.

2 Definitions

Stable Sort — a sort where elements with equal keys retain their original relative order after sorting.
In-place Sort — a sort that requires only O(1) (constant) extra memory, rearranging elements within the original array.
Divide and Conquer — an algorithmic strategy that breaks a problem into smaller subproblems, solves them recursively, and combines the results — used by Merge Sort and Quick Sort.

5 Real-Life Examples & Key Points

Exam Rankings: sorting scores to determine class rank — stability matters if ties should keep submission order.
E-commerce: sorting products by price, rating, or relevance — often uses Timsort (Python's built-in, a Merge+Insertion hybrid).
Nearly-Sorted Logs: new log entries appended to an already-sorted list — Insertion Sort shines here (adaptive).
Huge Datasets: sorting data too large for memory — External Merge Sort processes it in chunks.

6 Important Points

  • No comparison-based sort can beat O(n log n) in the worst case — this is a proven lower bound.
  • Merge Sort guarantees O(n log n) always, but needs O(n) extra space. Quick Sort is usually faster in practice but risks O(n²).
  • Non-comparison sorts (Counting, Radix) can beat O(n log n) — but only work for specific data types (e.g. integers in a known range).
  • Python's built-in sort() uses Timsort — a hybrid of Merge Sort and Insertion Sort, O(n log n) worst case, O(n) best case.

8 Python Code Examples

Bubble Sort

def bubble_sort(arr): n = len(arr) for i in range(n): for j in range(0, n-i-1): if arr[j] > arr[j+1]: arr[j], arr[j+1] = arr[j+1], arr[j] return arr

Merge Sort

def merge_sort(arr): if len(arr) <= 1: return arr mid = len(arr) // 2 left = merge_sort(arr[:mid]) right = merge_sort(arr[mid:]) return merge(left, right) def merge(left, right): result, i, j = [], 0, 0 while i < len(left) and j < len(right): if left[i] <= right[j]: result.append(left[i]); i += 1 else: result.append(right[j]); j += 1 return result + left[i:] + right[j:]

Quick Sort

def quick_sort(arr): if len(arr) <= 1: return arr pivot = arr[len(arr)//2] left = [x for x in arr if x < pivot] mid = [x for x in arr if x == pivot] right = [x for x in arr if x > pivot] return quick_sort(left) + mid + quick_sort(right)

9 Tips & Tricks

"Guaranteed O(n log n), space isn't a concern" → Merge Sort. "Fast in practice, space-constrained" → Quick Sort / Heap Sort.
"Nearly sorted / small input" → Insertion Sort (adaptive, beats O(n log n) sorts for tiny/nearly-sorted data).
"Integers in a small known range" → Counting Sort can be O(n), beating comparison-sort limits.

10 Common Mistakes

Assuming Quick Sort is always O(n log n) — worst case (already-sorted input + bad pivot choice) is O(n²).
Forgetting that Selection Sort and Quick Sort are typically unstable — equal elements may get reordered.
Using Counting/Radix Sort on non-integer or unbounded-range data — they don't generalise like comparison sorts.

        14 Chapter Summary

        Comparison-based sorts (Bubble, Selection, Insertion) are simple but O(n²). Merge Sort guarantees O(n log n) with O(n) space; Quick Sort is fast in practice (O(n log n) average) but risks O(n²) worst case; Heap Sort guarantees O(n log n) with O(1) space. Counting/Radix Sort beat the O(n log n) comparison-sort lower bound for specific data types.
