Stacks & Queues

1 Learning Objectives

  • Explain the LIFO principle of Stacks and FIFO principle of Queues.
  • Perform push, pop, enqueue, dequeue and analyse their time complexity.
  • Understand Circular Queues and Deques.
  • Apply Stacks to balanced-parentheses checking and expression evaluation.
  • Apply Queues to BFS-style processing and task scheduling.
  • Implement Stack/Queue operations in Python from scratch.

2 Definitions

Stack — a linear data structure that follows LIFO (Last In, First Out) — the most recently added element is removed first.
Queue — a linear data structure that follows FIFO (First In, First Out) — the earliest added element is removed first.
Deque — a Double-Ended Queue — allows insertion and deletion from both the front and the rear.

3 Concepts — Stack, Queue, Circular Queue, Deque

Stack — LIFO

Plate 1
Plate 2
Plate 3 ← top
push() adds to top, pop() removes from top

Queue — FIFO

Front → P1
P2
P3 ← Rear
enqueue() adds to rear, dequeue() removes from front

Circular Queue

A regular array-based Queue wastes space at the front after dequeues. A Circular Queue wraps the rear pointer back to index 0 once it reaches the end, reusing freed slots.

Deque (Double-Ended Queue)

Allows push/pop from both front and rear — a generalisation of both Stack and Queue.

5 Real-Life Examples

Stack: browser back button, undo in a text editor, function call stack during recursion, balanced-bracket checking.
Queue: printer job queue, ticket counter line, BFS traversal, CPU round-robin scheduling.
Deque: sliding window maximum problems, browser tab history (both directions).
Priority Queue: hospital emergency room triage — most critical patient served first, not by arrival order.

6 Important Points

  • Stack: push/pop/peek all O(1). Queue: enqueue/dequeue/front all O(1) (with proper implementation).
  • A naive array Queue wastes space — use a Circular Queue or a Linked-List-based Queue instead.
  • Any Queue can be built from two Stacks, and any Stack from two Queues — a classic interview question.
  • Recursion = an implicit Stack. Deep recursion can cause a real Stack Overflow.

8 Python Code Examples

Stack using a Python list

class Stack: def __init__(self): self.items = [] def push(self, x): self.items.append(x) # O(1) def pop(self): return self.items.pop() # O(1) def peek(self): return self.items[-1]

Queue using collections.deque

from collections import deque q = deque() q.append(10) # enqueue O(1) q.append(20) q.popleft() # dequeue O(1)

Balanced Parentheses Check

def is_balanced(expr): stack = [] pairs = {')': '(', ']': '[', '}': '{'} for ch in expr: if ch in '([{': stack.append(ch) elif ch in pairs: if not stack or stack.pop() != pairs[ch]: return False return not stack

9 Tips & Tricks

Use Python's collections.deque for Queues — it's O(1) at both ends, unlike a plain list's pop(0) which is O(n).
"Balanced brackets", "expression evaluation", "next greater element" → think Stack immediately.
"Level-order traversal", "shortest path (unweighted)" → think Queue (BFS) immediately.

10 Common Mistakes

Using Python list's pop(0) for Queue dequeue — this is O(n), not O(1)! Use deque.popleft() instead.
Forgetting to check for empty Stack/Queue before pop/dequeue — causes runtime errors.
Confusing Stack (LIFO) and Queue (FIFO) behaviour when converting between them.

        14 Chapter Summary

        A Stack (LIFO) supports O(1) push/pop/peek — perfect for undo, recursion, and expression evaluation. A Queue (FIFO) supports O(1) enqueue/dequeue — perfect for scheduling and BFS. Circular Queues fix array-Queue space wastage; Deques generalise both. Always use collections.deque in Python for efficient Queue operations.

        15 Key Formulae

        Stack: LIFO — last pushed, first popped
        Queue: FIFO — first enqueued, first dequeued
