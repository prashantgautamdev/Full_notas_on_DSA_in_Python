Chapter 3

Linked Lists

1 Learning Objectives

  • Explain what a Linked List is and how nodes connect via pointers.
  • Differentiate Singly, Doubly, and Circular Linked Lists.
  • Analyse time complexity of insertion, deletion, search and traversal.
  • Compare Linked Lists vs Arrays — when to use which.
  • Master the slow/fast pointer technique for cycle detection and middle-node problems.
  • Implement basic Linked List operations in Python.

2 Definitions

Linked List — a linear data structure where each element (node) stores data plus a pointer/reference to the next node — nodes are scattered in memory, not contiguous.
Node — the basic unit of a Linked List containing a data field and one or more pointer fields (next, and prev for doubly linked lists).
Head — a pointer to the first node of the Linked List — the entry point for all traversal.

Each node has an extra prev pointer — allows backward traversal at the cost of extra memory per node.

Circular Linked List

The last node's next points back to the head instead of NULL — useful for round-robin scheduling, playlists on loop, etc.

5 Real-Life Examples

Music Playlist: next/previous song — a Doubly Linked List fits perfectly.
Browser History: back/forward navigation using a Doubly Linked List.
Train Coaches: coaches (nodes) connected end-to-end — add/remove without shifting others.
Round-Robin Scheduler: a Circular Linked List cycles through processes fairly.

6 Important Points

  • Linked Lists trade O(1) index access (which they don't have) for O(1) insert/delete at head.
  • Every node needs extra memory for the pointer(s) — a real memory overhead vs arrays.
  • The slow/fast pointer technique elegantly solves middle-node and cycle-detection problems in O(n) time, O(1) space.
  • Always handle edge cases: empty list, single node, and operations on head/tail.

8 Python Code Examples

Node Class & Basic Structure

class Node: def __init__(self, data): self.data = data self.next = None

Insert at Head

# O(1) def insert_at_head(head, data): new_node = Node(data) new_node.next = head return new_node # new head

Reverse a Linked List

# O(n) time, O(1) space def reverse_list(head): prev = None curr = head while curr: nxt = curr.next curr.next = prev prev = curr curr = nxt return prev # new head

Detect Cycle (Floyd's Algorithm)

# O(n) time, O(1) space def has_cycle(head): slow = fast = head while fast and fast.next: slow = slow.next fast = fast.next.next if slow == fast: return True return False

9 Tips & Tricks

Always draw the pointers on paper before coding — Linked List bugs are almost always pointer-order mistakes.
Use a Dummy/Sentinel node before the head to avoid special-casing "empty list" or "delete head" scenarios.
Whenever you see "middle node", "cycle", or "nth from end" — think slow/fast pointers immediately.

10 Common Mistakes

Losing the reference to the rest of the list when reversing (forgetting to save 'next' before overwriting it).
Not handling the empty-list or single-node edge cases separately.
Creating a cycle accidentally while manipulating pointers (e.g. during reversal) and causing infinite loops.

        14 Chapter Summary

        A Linked List stores data in nodes connected by pointers, trading O(1) index access for flexible O(1) head insert/delete. Singly lists move forward only; Doubly lists move both ways at the cost of extra memory; Circular lists loop back to the head. The slow/fast pointer pattern is the single most powerful trick for Linked List problems — middle node, cycle detection, and nth-from-end all use it.

        15 Key Formulae

        Fast pointer moves 2 steps, Slow pointer moves 1 step → they meet inside a cycle (Floyd's Cycle Detection)
