2 Chapter 2 — Arrays

ConceptKey Takeaway
ArrayContiguous memory, indexed access
Access by indexO(1)
Search (unsorted)O(n)
Insert/Delete at endO(1) amortised
Insert/Delete at beginning/middleO(n)
Two-Pointer / Sliding Window / Prefix SumCore array techniques, O(n)
Golden Rule: Index access = O(1), everything else usually O(n) unless you use a technique.
pg. 4

3 Chapter 3 — Linked Lists

ConceptKey Takeaway
Linked ListNodes + pointers, scattered memory
Access by indexO(n)
Insert/Delete at headO(1)
Slow/Fast PointersMiddle node, cycle detection, nth from end
Floyd's AlgorithmCycle detection O(n)/O(1)
Golden Rule: Need fast insert/delete at ends? → Linked List. Need fast random access? → Array.

4 Chapter 4 — Stacks & Queues

ConceptKey Takeaway
StackLIFO — push/pop/peek O(1)
QueueFIFO — enqueue/dequeue O(1)
Circular QueueReuses freed slots
DequeInsert/delete both ends
Balanced Parens → StackBFS → Queue
Golden Rule: Undo/recursion/expression → Stack. Scheduling/BFS/ordering → Queue.

5 Chapter 5 — Recursion

ConceptKey Takeaway
RecursionFunction calls itself on smaller input
Base CaseMandatory stopping condition
Call StackRisk of Stack Overflow on deep recursion
MemoizationCache to avoid recomputation
BacktrackingRecursion + undo failed choices
Golden Rule: Overlapping subproblems? → Memoize. Trying & undoing choices? → Backtrack.
pg. 7

6 Chapter 6 — Trees

ConceptKey Takeaway
TreeHierarchical, one root, no cycles
BSTLeft < Node < Right
Balanced BSTO(log n) ops
Skewed BSTDegrades to O(n)
In-order traversalSorted output from a BST
Golden Rule: Need sorted order? → In-order traversal. Need O(log n) guarantee? → self-balancing tree.
pg. 8

7 Chapter 7 — Graphs

ConceptKey Takeaway
GraphVertices + Edges
Adjacency ListO(V+E) space, sparse graphs
Adjacency MatrixO(V²) space, dense graphs
BFSShortest path, unweighted
Dijkstra'sShortest path, weighted, non-negative
Golden Rule: Unweighted shortest path → BFS. Weighted (non-neg) → Dijkstra. Negative weights → Bellman-Ford.
pg. 9

8 Chapter 8 — Sorting Algorithms

ConceptKey Takeaway
Merge SortO(n log n) always, O(n) space, stable
Quick SortO(n log n) avg, O(n²) worst
Heap SortO(n log n) always, O(1) space
Comparison-sort lower boundΩ(n log n)
Counting/Radix SortBeats O(n log n) for bounded integers
Golden Rule: Guaranteed O(n log n)+stable → Merge Sort. Fast+low memory → Quick/Heap Sort. Bounded ints → Counting/Radix.
pg. 10

9 Chapter 9 — Searching Algorithms

ConceptKey Takeaway
Linear SearchO(n), any data
Binary SearchO(log n), needs sorted data
Search on AnswerBinary Search over a monotonic predicate
1M elementsLinear: 1M steps | Binary: ~20 steps
Golden Rule: Sorted data? → Binary Search. Unsorted+frequent lookups? → Hash Table.

10 Chapter 10 — Hashing

ConceptKey Takeaway
Hash TableAverage O(1) insert/lookup/delete
CollisionTwo keys, same index
Chaining vs Open AddressingTwo collision-resolution strategies
Load Factorn / table_size, triggers rehashing
Two Sum patternO(n²) → O(n) with a Hash Map
Golden Rule: Need O(1) lookup/duplicate-check, don't care about order → Hash Table/Set.
