Graphs

2 Definitions

Graph — a non-linear data structure consisting of a set of vertices (nodes) connected by edges — can be directed or undirected, weighted or unweighted.
Adjacency List — a graph representation where each vertex stores a list of its directly connected neighbours — space-efficient for sparse graphs.
Adjacency Matrix — a V×V matrix where cell [i][j] = 1 (or weight) if an edge exists between vertex i and j — O(1) edge lookup but O(V²) space.

5 Real-Life Examples

Social Network: people = vertices, friendships = edges (undirected).
Google Maps: intersections = vertices, roads = weighted edges (distance/time).
Course Prerequisites: a Directed Acyclic Graph — solved via Topological Sort.
Web Pages: pages = vertices, hyperlinks = directed edges.

6 Important Points

  • Use Adjacency List for sparse graphs (most real-world graphs); use Adjacency Matrix for dense graphs or O(1) edge lookups.
  • BFS guarantees shortest path in an unweighted graph; for weighted graphs, use Dijkstra's Algorithm.
  • Topological Sort only works on a DAG (Directed Acyclic Graph) — if there's a cycle, no valid ordering exists.
  • Always track visited nodes during BFS/DFS to avoid infinite loops in cyclic graphs.

8 Python Code Examples

Graph as Adjacency List

graph = { 'A': ['B', 'C'], 'B': ['A', 'D'], 'C': ['A'], 'D': ['B'], }

BFS Traversal

from collections import deque def bfs(graph, start): visited = {start} queue = deque([start]) order = [] while queue: node = queue.popleft() order.append(node) for neighbour in graph[node]: if neighbour not in visited: visited.add(neighbour) queue.append(neighbour) return order

DFS Traversal (Recursive)

def dfs(graph, node, visited=None): if visited is None: visited = set() visited.add(node) for neighbour in graph[node]: if neighbour not in visited: dfs(graph, neighbour, visited) return visited

9 Tips & Tricks

"Shortest path, unweighted graph" → BFS. "Shortest path, weighted graph" → Dijkstra.
"Dependency ordering / prerequisites" → Topological Sort (only works on a DAG).
Always initialise a visited set — forgetting it is the #1 cause of infinite loops in graph code.

10 Common Mistakes

Forgetting to mark nodes as visited, causing infinite loops in cyclic graphs.
Using Dijkstra's Algorithm on a graph with negative edge weights — it gives incorrect results; use Bellman-Ford instead.
Confusing Adjacency Matrix O(V²) space with Adjacency List O(V+E) space, leading to poor choices for sparse graphs.

        14 Chapter Summary

        A Graph models vertices connected by edges — directed/undirected, weighted/unweighted. Represent sparse graphs with an Adjacency List (O(V+E) space), dense graphs with an Adjacency Matrix (O(V²) space). BFS (Queue) finds shortest paths in unweighted graphs; DFS (Stack/recursion) is great for cycle detection and topological sort. For weighted shortest paths, use Dijkstra's (non-negative weights) or Bellman-Ford (handles negatives).

        15 Key Formulae

        BFS/DFS: O(V + E)  |  Dijkstra (heap): O((V+E) log V)  |  Bellman-Ford: O(V × E)
