Chapter 5

Recursion

1 Learning Objectives

  • Understand what Recursion is and identify Base Case vs Recursive Case.
  • Trace the Call Stack and recursion tree for simple functions.
  • Differentiate Head vs Tail recursion.
  • Apply Memoization to optimise exponential recursive solutions.
  • Understand Backtracking as an extension of recursion.
  • Convert between recursive and iterative solutions.

2 Definitions

Recursion — a technique where a function calls itself, directly or indirectly, to solve smaller instances of the same problem.
Base Case — the condition under which a recursive function stops calling itself and returns a direct answer — without a base case, recursion never terminates.
Call Stack — the internal stack the program uses to keep track of function calls — each recursive call pushes a new frame, and returning pops it.

3 Concepts — Recursion Tree, Head vs Tail

Anatomy of a Recursive Function

function(n):
  if base_case(n): return direct_answer
  else: return combine(n, function(smaller_n))

Recursion Tree — Fibonacci(4)

Notice fib(2) is computed twice — this overlapping subproblem is exactly why naive recursive Fibonacci is O(2ⁿ), and why Memoization helps.

Head vs Tail Recursion

Head RecursionTail Recursion
Recursive call happens firstRecursive call is the last statement
Processing happens after the call returnsNo pending work after the call
Cannot be optimised by most compilersCan be optimised to O(1) space (not in Python though!)

5 Real-Life Examples

Nested Folders: opening a folder that contains folders that contain folders — recursion naturally mirrors this.
Russian Nesting Dolls: open one, find a smaller one inside, repeat until you hit the smallest (base case).
Family Tree: "find my ancestor" naturally recurses up the tree until reaching the root.
Tower of Hanoi: move n-1 disks, move the largest, move n-1 disks again — a textbook recursive problem.

6 Important Points

  • Every recursive function needs at least one Base Case — this is the #1 source of bugs.
  • Recursion uses the Call Stack implicitly — deep recursion can cause a Stack Overflow.
  • Memoization (caching results) turns exponential recursive solutions into polynomial ones.
  • Backtracking = recursion + "undo the choice" when a path doesn't work (used in maze solving, N-Queens, permutations).

8 Python Code Examples

Factorial (simple recursion)

def factorial(n): if n <= 1: # base case return 1 return n * factorial(n - 1) # recursive case

Fibonacci — Naive vs Memoized

# Naive: O(2^n) def fib(n): if n <= 1: return n return fib(n-1) + fib(n-2) # Memoized: O(n) def fib_memo(n, memo={}): if n <= 1: return n if n in memo: return memo[n] memo[n] = fib_memo(n-1, memo) + fib_memo(n-2, memo) return memo[n]

Tower of Hanoi

def hanoi(n, src, aux, dst): if n == 0: return hanoi(n-1, src, dst, aux) print(f"Move disk {n} from {src} to {dst}") hanoi(n-1, aux, src, dst)

9 Tips & Tricks

Always write the base case first when designing a recursive function — it defines when the recursion stops.
If you see "overlapping subproblems" in the recursion tree, reach for Memoization (a dict cache) immediately.
To trace a recursive function, draw the recursion tree on paper — it reveals both correctness and complexity.

10 Common Mistakes

Forgetting the base case entirely, or writing one that's never reached — causes infinite recursion / Stack Overflow.
Using a mutable default argument (like memo={}) carelessly across multiple calls without understanding Python's default-argument behaviour.
Assuming recursion is always slower — for problems without overlapping subproblems, it's just as efficient as iteration.

        14 Chapter Summary

        Recursion solves a problem by breaking it into smaller versions of itself, until a Base Case is hit. It uses the Call Stack implicitly, so deep recursion risks Stack Overflow. When subproblems overlap (like Fibonacci), Memoization turns exponential time into polynomial time. Backtracking extends recursion by "undoing" failed choices — essential for maze-solving, permutations, and N-Queens.

        15 Key Formulae

        T(n) = T(n-1) + O(1) → O(n) [linear recursion]
        T(n) = 2·T(n-1) + O(1) → O(2ⁿ) [branching recursion, e.g. naive Fibonacci]
