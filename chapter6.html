Trees

2 Definitions

Tree — a hierarchical, non-linear data structure made of nodes connected by edges, with one root node and no cycles.
Binary Search Tree (BST) — a Binary Tree where, for every node, all values in the left subtree are smaller and all values in the right subtree are larger.
Height of a Tree — the number of edges on the longest path from the root to a leaf node.

3 Concepts — Terminology, BST, Traversals

Tree Terminology

Root = topmost node. Leaf = node with no children. Height = longest root-to-leaf path (here, 2 edges).

Binary Search Tree (BST) Property

For every node: left subtree values < node value < right subtree values

Traversal Orders

TraversalOrderUse Case
In-orderLeft → Root → RightGet sorted order from a BST
Pre-orderRoot → Left → RightCopy/serialize a tree
Post-orderLeft → Right → RootDelete a tree safely
Level-orderLevel by level (uses Queue)Shortest-path-like / BFS problems

5 Real-Life Examples

Org Chart: CEO at the root, managers as children, employees as leaves.
File System: drives → folders → subfolders → files, a natural tree.
Database Index (B-Tree): allows fast lookups on disk-based data at massive scale.
Autocomplete (Trie): each path from root spells out a prefix of stored words.

6 Important Points

  • A balanced BST gives O(log n) search/insert/delete; a skewed (degenerate) BST degrades to O(n) — basically a Linked List!
  • In-order traversal of a BST always yields elements in sorted order — a very handy property.
  • AVL Trees and Red-Black Trees self-balance to guarantee O(log n) even in the worst case.
  • Trees are the base for many advanced structures: Heaps, Tries, Segment Trees, B-Trees.

8 Python Code Examples

Tree Node & BST Insert

class Node: def __init__(self, val): self.val = val self.left = self.right = None def insert(root, val): if root is None: return Node(val) if val < root.val: root.left = insert(root.left, val) else: root.right = insert(root.right, val) return root

In-order Traversal

def inorder(root, result=None): if result is None: result = [] if root: inorder(root.left, result) result.append(root.val) inorder(root.right, result) return result # sorted order!

Height of Tree

def height(root): if root is None: return -1 return 1 + max(height(root.left), height(root.right))

9 Tips & Tricks

"Sorted output needed" → think in-order traversal of a BST immediately.
Most Tree problems (height, diameter, balance-check) are solved with post-order style recursion — process children first, then combine.
Always ask: "is this tree guaranteed balanced?" — if not, worst-case complexity becomes O(n), not O(log n).

10 Common Mistakes

Assuming BST operations are always O(log n) — they're only O(log n) if the tree is balanced!
Confusing 'height' (edges to deepest leaf) with 'number of nodes' when computing complexity.
Forgetting the null/None check as the base case in recursive tree functions, causing crashes on empty subtrees.

        14 Chapter Summary

        A Tree is a hierarchical, non-linear structure with one root and no cycles. A BST keeps left < node < right, giving O(log n) operations — if balanced. In-order traversal yields sorted output. AVL/Red-Black Trees self-balance to guarantee worst-case O(log n). Trees underpin Heaps, Tries, and database B-Trees.
