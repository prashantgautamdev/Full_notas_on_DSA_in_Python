Chapter 9

Searching Algorithms

1 Learning Objectives

  • Understand Linear Search and Binary Search and their preconditions.
  • Analyse time complexity: O(n) vs O(log n) and why it matters at scale.
  • Learn Jump, Interpolation, and Exponential Search variants.
  • Apply Binary Search to rotated arrays and "search on answer" optimisation problems.
  • Implement iterative and recursive Binary Search in Python.
  • Choose the right search technique for a given real-world scenario.

2 Definitions

Linear Search — a search technique that checks every element one by one until the target is found or the array ends — O(n) time, works on unsorted data.
Binary Search — a search technique that repeatedly halves the search space by comparing the target to the middle element — O(log n) time, requires sorted data.
Search Space — the range of possible positions/values being considered at any point during a search algorithm's execution.

5 Real-Life Examples & Key Points

Dictionary: flipping to roughly the right page and narrowing — a real-life Binary Search!
Phone Contact List: sorted alphabetically, Binary Search finds a name in O(log n).
"Guess the Number" Game: "higher or lower" hints let you binary-search the answer in ~log₂(n) guesses.
Debugging with Git Bisect: finding which commit introduced a bug using binary search over commit history!

6 Important Points

  • Binary Search requires sorted data — this is the #1 precondition to check before using it.
  • Binary Search isn't just for arrays — it generalises to "search on answer" problems with any monotonic predicate.
  • Interpolation Search can beat Binary Search on uniformly distributed data, but degrades badly on skewed data.
  • For unsorted data with frequent lookups, consider a Hash Table instead — O(1) average lookup, no sorting needed.

8 Python Code Examples

Linear Search

def linear_search(arr, target): for i, val in enumerate(arr): if val == target: return i return -1 # O(n)

Binary Search (Iterative)

def binary_search(arr, target): lo, hi = 0, len(arr) - 1 while lo <= hi: mid = (lo + hi) // 2 if arr[mid] == target: return mid elif arr[mid] < target: lo = mid + 1 else: hi = mid - 1 return -1 # O(log n)

Search in Rotated Sorted Array

def search_rotated(arr, target): lo, hi = 0, len(arr) - 1 while lo <= hi: mid = (lo + hi) // 2 if arr[mid] == target: return mid if arr[lo] <= arr[mid]: # left half sorted if arr[lo] <= target < arr[mid]: hi = mid - 1 else: lo = mid + 1 else: # right half sorted if arr[mid] < target <= arr[hi]: lo = mid + 1 else: hi = mid - 1 return -1

9 Tips & Tricks

Whenever you see "sorted array" in a problem, your brain should immediately think Binary Search.
"Find minimum X such that condition holds" → Binary Search on Answer, even if there's no array involved!
To avoid integer overflow in other languages, compute mid as lo + (hi - lo) // 2 instead of (lo + hi) // 2 (not an issue in Python, but good habit).

10 Common Mistakes

Applying Binary Search to unsorted data — it silently gives wrong answers instead of erroring.
Off-by-one errors in the lo/hi boundary updates, causing infinite loops or missed elements.
Forgetting to handle duplicate elements when searching for "first" or "last" occurrence.

        14 Chapter Summary

        Linear Search (O(n)) works on any data but doesn't scale. Binary Search (O(log n)) requires sorted data but is dramatically faster at scale. Variants like Jump, Interpolation, and Exponential Search trade off preconditions for speed in specific scenarios. Binary Search generalises beyond arrays to any problem with a monotonic predicate — "search on answer".

        15 Key Formulae

        Binary Search comparisons ≈ log₂(n)  |  Jump Search block size = √n
