Arrays

2 Definitions

Array — a linear data structure that stores elements of the same type in contiguous memory locations, accessed using an index.
Static Array — an array whose size is fixed at creation time and cannot grow or shrink.
Dynamic Array — an array that automatically resizes (usually by doubling) when it runs out of capacity — e.g. Python's list.

3 Concepts — Memory Layout, Static vs Dynamic, 2D Arrays

Memory Layout

Because array elements sit in contiguous memory, the address of any element can be computed instantly:

address(arr[i]) = base_address + i × size_of(element)

This is exactly why arr[i] access is O(1) — no traversal needed, just a direct address calculation!

Static vs Dynamic Arrays

Static ArrayDynamic Array
Fixed size, set onceGrows automatically (usually doubles)
C arrays, Java arraysPython list, C++ vector, Java ArrayList
No resizing overheadOccasional O(n) resize, but O(1) amortised append
"Amortised O(1)" means: most appends are O(1), and the occasional expensive O(n) resize gets "averaged out" over many operations — so on average it still behaves like O(1).

2D Arrays

A 2D array is an "array of arrays" — think of a grid, like a tic-tac-toe board.

123
456
789

Accessed as arr[row][col] — stored in memory using Row-Major Order (row by row) in most languages including Python and C.

5 Real-Life Examples

Class Marksheet: an array of 50 integers storing marks — index = roll number, value = marks.
Tic-Tac-Toe Board: a 2D array (3×3) storing 'X', 'O', or empty at each cell.
Image Pixels: a 2D (or 3D with RGB channels) array where each cell is a pixel's colour value.
Leaderboard: an array of scores that gets sorted to show the top players.

6 Important Points

  • Array access by index is always O(1) — the single biggest advantage over Linked Lists.
  • Insertion/deletion at the beginning or middle is O(n) due to shifting.
  • A Dynamic Array trades a little memory overhead for flexible resizing.
  • 2D arrays use Row-Major Order in Python/C — this affects cache performance.

8 Python Code Examples

Array Traversal & Sum

# O(n) traversal to compute sum def array_sum(arr): total = 0 for x in arr: total += x return total

Reverse Array In-Place (Two-Pointer)

# O(n) time, O(1) extra space def reverse_array(arr): left, right = 0, len(arr) - 1 while left < right: arr[left], arr[right] = arr[right], arr[left] left += 1 right -= 1 return arr

Rotate Array Left by k (Reversal Algorithm)

# O(n) time, O(1) extra space def rotate_left(arr, k): n = len(arr) k = k % n arr[:] = arr[k:] + arr[:k] return arr

Prefix Sum Array

# O(n) preprocessing -> O(1) range-sum queries def build_prefix_sum(arr): prefix = [0] * (len(arr) + 1) for i in range(len(arr)): prefix[i+1] = prefix[i] + arr[i] return prefix # sum(l, r) = prefix[r+1] - prefix[l]

9 Tips & Tricks

Whenever a problem says "find pair/subarray with sum X" — think Two-Pointer (sorted array) or Hash Set (unsorted array) before brute force.
"Rotate array" and "reverse array" problems almost always solve elegantly using the reversal trick: reverse whole → reverse parts.
If a problem needs repeated range-sum queries, precompute a Prefix Sum array once — turns O(n) per query into O(1).

10 Common Mistakes

Forgetting that Python's list is dynamic — assuming append() is always instantly O(1) with zero cost, ignoring occasional resize.
Off-by-one errors when computing prefix sums or sliding window boundaries.
Confusing row-major vs column-major indexing when working with 2D arrays / matrices.
